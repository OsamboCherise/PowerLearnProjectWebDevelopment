Semantic HTML5 Elements

Why Semantic HTML Matters

Semantic HTML provides meaning to web content, making it more accessible and improving SEO. By using elements that describe their purpose, both browsers and developers can better understand the structure of web pages.

<header>
  <h1>Website Title</h1>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>
</main>

<footer>
  <p>Copyright information</p>
</footer>

Common Semantic Elements

<header>

Represents introductory content, typically a group of introductory or navigational aids.

<nav>

Defines a set of navigation links.

<main>

Specifies the main content of a document.

<article>

Defines self-contained content that could exist independently.

<section>

Defines a section in a document.

<footer>

Represents a footer for its nearest sectioning content or root element.

Web Accessibility Guidelines

Importance of Accessibility

Web accessibility ensures that people with disabilities can perceive, understand, navigate, and interact with websites and tools. It's not just a best practiceâ€”in many cases, it's the law.

Key Accessibility Techniques

  • Provide text alternatives for non-text content
  • Create content that can be presented in different ways
  • Make all functionality available from a keyboard
  • Help users navigate and find content
  • Make text readable and understandable

ARIA Landmarks and Roles

ARIA (Accessible Rich Internet Applications) defines ways to make web content more accessible:

<div role="navigation" aria-label="Main menu">
  <!-- navigation items -->
</div>

<button aria-expanded="false" aria-controls="dropdown-menu">
  Menu
</button>
<ul id="dropdown-menu" role="menu" hidden>
  <li role="menuitem"><a href="#home">Home</a></li>
  <li role="menuitem"><a href="#about">About</a></li>
</ul>

SEO Best Practices

On-Page SEO Techniques

Search Engine Optimization helps your website rank higher in search results, bringing more organic traffic.

  1. Use descriptive, keyword-rich titles and headings
  2. Include meta descriptions for each page
  3. Optimize images with alt text
  4. Create a logical URL structure
  5. Ensure fast loading times
  6. Make your site mobile-friendly

Structured Data

Structured data helps search engines understand your content better, which can lead to rich results and improved visibility.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Article headline",
  "author": {
    "@type": "Person",
    "name": "Author Name"
  },
  "datePublished": "2023-01-01",
  "description": "Article description"
}
</script>

Responsive Web Design

Mobile-First Approach

With mobile devices accounting for over half of web traffic, a mobile-first approach ensures your website works well on all screen sizes.

/* Base styles for mobile */
.container {
  padding: 15px;
  margin: 0 auto;
}

/* Tablet styles */
@media (min-width: 768px) {
  .container {
    padding: 30px;
    max-width: 90%;
  }
}

/* Desktop styles */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
  }
}

Web Performance Optimization

Key Performance Metrics

Website speed directly impacts user experience and conversion rates. Important metrics include:

  • First Contentful Paint (FCP)
  • Largest Contentful Paint (LCP)
  • First Input Delay (FID)
  • Cumulative Layout Shift (CLS)

Performance Optimization Techniques

Minification

Minify CSS, JavaScript, and HTML to reduce file sizes.

Compression

Use Gzip or Brotli compression to reduce transfer size.

Caching

Implement proper caching strategies for static assets.

Lazy Loading

Load images and other media only when they're needed.
